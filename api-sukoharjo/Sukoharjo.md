
# SECURITY AUDIT REPORT: Broken Authentication & Information Disclosure

**Target:** `https://api-dev.sukoharjokab.go.id`

**Researcher:** ZDN

**Severity:** **CRITICAL**

**Date:** April 26, 2026

---

## 1. Executive Summary

Ditemukan celah keamanan kritis pada platform Satudata Kabupaten Sukoharjo (server development). Celah ini memungkinkan pihak luar untuk mendaftarkan akun dengan hak akses **Administrator** tanpa verifikasi, yang kemudian dapat digunakan untuk mengakses data sensitif seluruh staf internal organisasi pemerintah yang terdaftar di sistem.

---

## 2. Vulnerability Detail

### A. Broken Authentication (CWE-287)

Endpoint `/auth/register` pada server API terbuka untuk publik walaupun disallow method GET tanpa adanya pembatasan atau verifikasi admin. Hal ini memungkinkan siapa saja membuat akun dengan parameter `role_id: 1` (Administrator).

### B. Sensitive Information Disclosure (CWE-200)

Setelah mendapatkan akses Administrator, penyerang dapat mengakses endpoint `/user` banyak seperti yang dapat user lakukan seperti membocorkan data pribadi staf internal, termasuk:

- Nama Lengkap & Username
- Email Dinas (`@sukoharjokab.go.id`)
- Struktur Organisasi (Diskominfo, BPS, Bapperida, dll.)
- Role ID dan UUID Internal.
Dan juga dapat memanipulasi post,data dll yang berhubungan dengaan administrator

## 5. Rekomendasi Perbaikan

1. **Menutup Endpoint Register**: Menonaktifkan fungsi registrasi publik pada server produksi maupun development. Pembuatan akun harus melalui panel admin yang sudah ada.
2. **Implementasi Middleware**: Memastikan setiap endpoint sensitif (seperti `/user`) dilindungi oleh middleware yang mengecek validitas token dan level otorisasi secara ketat.
3. **Data Masking**: Menyembunyikan informasi sensitif pada respon API jika tidak diperlukan secara fungsional.
